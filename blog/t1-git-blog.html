Git and GitHub

Saviors of Version Control

August 24, 2015

Imagine you and a group coding away in order to create a beautiful, meaningful project when all of a sudden, a member accidentally overwrites a file. Now this file is lost and can never be retrieved. Version control helps prevent accidents like this! A version control system allows you to record snapshots of changes to a file or directory so that you can look back to specific versions in the future. Every change made is tracked, along with who made the change and why. You can think about it like a saving point in a video game. Version control is important to developers because you can always go back to the last saved version of your work if you make any mistakes or even accidentally delete your entire project. Version control not only helps the individual but it also helps with group projects. Group members working on a single project can use source control to communicate with eachother and make changes with confidence. They can also see who made a change, where, and why.

Git and GitHub are two very prominent forms of version control for programmers. Although they serve very similar purposes, Git and GitHub are not the same thing. Git is a version control software that is downloaded and lives locally on your computer. GitHub is also a form of version control, but lives remotely on the internet. Think of Git as a word processor and think of GitHub as Google Docs. Word processors only allows the user to access and edit your work locally, while Google Docs allow a team of individuals to simultaneously edit and improve a single project. You can store and access your work online, on GitHub, but only if you expicitly send it there, via the command "git push." Using GitHub also allows other programmers to see and use your code in their work, saving the entire programming community a lot of time. GitHub is amazing because it allows people to share their work and collaborate together on giant projects.
